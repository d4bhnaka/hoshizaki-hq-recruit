# 01. プロジェクト計画

## プロジェクト概要

- **名称**：ホシザキ株式会社 本社採用サイト
- **目的**：本社採用向けのコーポレートリクルーティングサイトを Astro で静的構築する。ブランドメッセージ「常識の先へ、跳べ。」を軸に、応募者へ会社の価値観・働く環境・社員の姿を伝える。
- **性質**：静的サイト。サーバーサイド処理はなく、ビルド成果物（HTML/CSS/JS/アセット）を FTP でサーバーへ配置する運用を想定。
- **納品形態**：クライアント（ホシザキ担当者）が納品後に**自ら HTML を直接編集して更新**する。このため成果物は圧縮されず、Astro 固有の属性や識別子が残らず、ファイルが分割されている必要がある。配置先のサブディレクトリが不定のため、**すべてのパスが相対**であることも必須。
- **Figma デザイン**：[トップページ (node 296:7137)](https://www.figma.com/design/Q5PQirN5wGl9c1AqJJHZBt/%E3%83%9B%E3%82%B7%E3%82%B6%E3%82%AD%E3%80%80%E6%9C%AC%E7%A4%BE%E6%8E%A1%E7%94%A8?node-id=296-7137&m=dev)

## 納品時に満たすべき要件（絶対条件）

1. `dist/` 以下の HTML に `data-astro-cid-*` など Astro 固有属性が**一切残っていない**こと。
2. HTML / CSS / JS が**圧縮されていない**（インデント保持）。
3. CSS は `css/style.css` として**外部ファイル化**されており、インライン化されていないこと。
4. JS／CSS／アセットの**ファイル名にハッシュが付かない**こと。
5. すべてのパスが**相対パス**で出力されていること（配置先のサブディレクトリが変わってもリンク切れしない）。
6. 下層ページは **`/about/index.html` 形式**（directory format）で出力されていること。

## リスクと前提

- **SP／タブレット向けデザインが部分的**：Figma 上で `top_sp`（`431:1181`, 402×4914）が確認できるのはトップのみ。他ページの SP デザインは未確認 → 各ページは当面 PC（1600px 基準）＋最低限のリキッド対応で実装し、SP 版が確定次第追記。
- **Figma Variables（カラー・タイポ・スペーシング）が未定義**：`get_variable_defs` は空。色・フォントサイズは各要素の Figma inspect から個別に抽出し、`foundation/_variables.scss` にトークン化していく方針。
- **Internship ページの正バージョン確認**：`246:815`（v1）と `446:5278`（v2）が並存。実装前に v2 が最新であることをデザイナーへ確認する。
- **パンくず専用 symbol の未確認**：Header バリアント 2（`361:539`）がパンくず入りの可能性あり。実装着手前に Figma 目視で確定させる。
- **アセット書き出し**：Figma から PNG／SVG／動画を書き出す作業が実装と並行して発生する。各ページ仕様書のアセット一覧を作業メモとして運用。
- **配置先ディレクトリが可変**：`base: "./"` を前提にすべて相対パス。**絶対パス参照（`/css/style.css` など）は禁止**。
