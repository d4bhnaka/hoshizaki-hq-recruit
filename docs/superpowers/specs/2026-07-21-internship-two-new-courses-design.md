# インターンシップ 新規2コース追加 設計

作成日: 2026-07-21
対象ページ: [src/pages/internship.astro](../../../src/pages/internship.astro)

## 目的

インターンシップページのコース一覧に、文理不問のWEB 1Dayコースを2件追加する。

- 【文理不問】1Day仕事体験　管理部門コース（WEB）
- 【文理不問】1Day仕事体験　経営戦略立案コース（WEB）

## 前提と制約

- コースは `src/pages/internship.astro` の `courses: Course[]` 配列で一元管理されている。カルーセル（CourseCard）、タブ切り替え、詳細セクション（CourseDetail）はすべてこの配列を回して生成されるため、**配列に要素を足すだけで全UIに反映される**。コンポーネントとSCSSの変更は不要。
- コース詳細の構造は `Course` 型（[src/components/CourseDetail.astro](../../../src/components/CourseDetail.astro)）に固定される。`id` / `title` / `image` / `points`(3件) / `program` / `programOverview` / `voices`(4件) をすべて埋める。
- 画像は `public/images/internship/` に配置し、`image` にはファイル名のみを渡す。
- タイトル表記は既存4件の書式に合わせる（`【対象】1Day仕事体験　コース名（WEB）`）。依頼原文の「１DAY」は既存が半角「1Day」で統一されているため半角に揃える。
- 本文はクライアント確認前の**仮原稿**である。管理部門コースのPOINT見出しとプログラム項目のみ依頼者指定、それ以外は既存コースのトーンに合わせたドラフト。

## 追加するデータ

配列末尾に `course-05`、`course-06` の順で追加する。既存4件は理系向け、新規2件は文理不問なので、理系→文理不問の並びになり導線として自然。

## ダミー画像

### 仕様

- サイズ: 720×540（既存 course-02 / course-04 と同じ）
- 形式: PNG、`public/images/internship/course-05.png` / `course-06.png`
- デザイン: ブランドのシアン（`#00c8ff` → `#00a0e9`）のグラデーション地に、白抜きで `DUMMY IMAGE` とコース名、差し替えるべき写真の内容メモを日本語で記載する

記載する内容メモ

- course-05: 経理の社員が数字データを使って仕事をしている様子
- course-06: オンラインで社員が学生と交流している様子

差し替え忘れが一目でわかることを最優先にしたデザインにする。写真らしく見せることはしない。

### 生成方法

スクラッチパッドにHTMLを1枚書き、ヘッドレスChrome（Playwright）で720×540のビューポートに描画してスクリーンショットを撮り、PNGとして書き出す。OSの日本語フォントがそのまま使えるため、フォントパスの指定や画像ライブラリの追加インストールが不要になる。生成に使ったHTMLはスクラッチパッドに置き、リポジトリにはコミットしない。

## 検証

1. `npm run build:scss && npx astro build` が通ること
2. dev サーバーでインターンシップページを開き、以下を目視確認する
   - カルーセルに6枚のカードが並び、矢印で新規2件まで到達できる
   - 新規カードをクリックすると対応する詳細セクションに切り替わる
   - 詳細内のPOINT・プログラム・開催概要・VOICEがレイアウト崩れなく表示される
   - ダミー画像がカードのサムネイル枠で正しくトリミングされる
3. 375px幅でも上記が崩れないこと
